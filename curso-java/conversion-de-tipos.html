Conversión de tipos

Muchas veces, necesitaremos operar con datos que tenemos almacenados en nuestras variables, pero podría ser que tuviéramos un problema al querer hacerlo al no coincidir con el tipo de dato.

en esta tabla podremos ver cómo se realizan las conversiones en Java:

Tipo destino
boolean char byte short int long float double
Tipo origen bolean - N N N N N N N
char N - C C CI CI CI CI
byte N C - CI CI CI CI CI
short N C C - CI CI CI CI
int N C C C - CI CI* CI
long N C C C C - CI* CI*
float N C C C C C - CI
float N C C C C C C -
  • N indica que no se puede convertir.
  • C indica que se tiene que convertir de manera explícita, es decir, debe ser el programador el que la realice.
  • CI indica que se realiza una conversión implícita, es decir, lo realiza Java implícitamente.
  • CI* indica que se realiza una conversión implícita pero podría haber una pérdida de datos.

Conversión de manera implícita

La conversión implícita la realiza de manera automática el intérprete cuando lea ésta. Hay que tener en cuenta que esto sólo será posible cuando se pase de un dato menos preciso a otro de mayor precisión.

Prácticamente no hay que hacer nada extraño, simplemente operar con ellos como si fueran el mismo tipo de dato.

También podríamos directamente copiar el valor de una variable de un tipo primitivo más pequeño a otro mayor realizando la conversión implícita.

Conversión de manera explícita

Este tipo conversión puede tener una pérdida de datos y somos nosotros los que tendremos que realizarlo de manera manual. Para ello, podremos orientarnos con la tabla de arriba cuándo será necesario realizarlo y cuándo no.
