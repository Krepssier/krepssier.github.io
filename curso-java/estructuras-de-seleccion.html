Una condicional en Java al igual que en otros lenguajes, nos realizará una tarea determinada si la condición se evalúa como verdadera. Es decir, se ejecutará lo que contenga el bloque de la estructura if que está delimitado por llaves. Mientras que si se evalúa como falso, no hará nada.

Operador lógico and &

Para que se muestre true, ambos operadores deben ser true, en caso contrario, devolverá false.

Operador lógico or |

Para que se muestre true, al menos uno de los dos operadores debe ser true, en caso contrario, devolverá false.

Operador lógico and perezoso &&

Es igual que &, pero si la primera evaluación es false, la segunda condición no se evaluará. Es por esto, que en la imagen que se muestra a continuación, el IDE nos está marcando una advertencia indicando que no se evaluará los booleanos donde inicialmente estaban en false. Y es de ahí que se les asigne el nombre de perezosos.

Operador lógico or perezoso ||

Funciona de manera similar al &&, pero aquí, el segundo no se evaluará si el primero es true.

Operador XOR lógico

Para que se evalúe como true, ambos booleanos deben ser contrarios

Operador lógico opuesto

Lo que hace este operador, es transformar el booleano a su contrario.

Operador de condicional

También es llamado operador ternario. Este operador evaluará una condición, si ésta aparece como true, se escogerá la instrucción que hay después de la interrogación y antes de los dos puntos, en caso contrario, la que va después de los dos puntos.

Estamos acostumbrados a ver las instrucciones para imprimir texto, pero se pueden utilizar para otras instrucciones, como para devolver un entero.

Condicional switch

Con switch podremos comparar varios valores, sin ser necesariamente booleanos. Es decir, podremos comparar texto, números, booleanos, caracteres..., y aquel que coincida, ejecutará el valor que tiene en su interior.

Entre paréntesis se introduce lo que queremos comparar. En cada case, se mostrará la opción que se compara. En cada bloque se terminará con break, ya que si no se hace, se ejecutarán los siguientes case aunque no se cumplan hasta encontrar el siguiente break.

Por último indicar que el default es opcional, pero es útil cuando no existe ninguna opción.
